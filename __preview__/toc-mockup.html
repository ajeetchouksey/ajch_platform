Shared Responsibility Model

AWS is responsible for the security of the cloud — physical hardware, networking, managed services. You are responsible for security in the cloud — OS patches, app code, IAM config, data encryption.

⚠️ Exam trap: Lambda = AWS manages OS patches. EC2 = you manage OS patches.

Customer responsibilities include: identity & access management, data classification, network traffic protection, OS & app configuration.

IAM — Identity & Access

IAM lets you control who can do what with which AWS resources. It is global (not region-specific) and free.

Users, Groups & Roles

A User is a person or service. A Group is a collection of users sharing the same permissions. A Role is assumed temporarily, typically by services or cross-account access.

Policies & Permissions

Policies are JSON documents attached to identities or resources. Default deny applies — explicit allow must exist. Deny always wins over Allow.

Network Security

Security Groups act as stateful firewalls at the instance level. NACLs are stateless and apply at the subnet level. WAF filters HTTP/S requests, Shield protects against DDoS.

Compliance Frameworks

AWS is compliant with hundreds of standards. Key ones for CCP: SOC 1/2/3, PCI DSS, HIPAA, ISO 27001. AWS Artifact provides on-demand access to compliance reports.
