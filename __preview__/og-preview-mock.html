How it works — Cloudflare Worker
The request flow:

1. LinkedIn crawler hits https://aaryaai.dev/blog/order-bot-domain-boundary
2. Cloudflare Worker intercepts — detects User-Agent: LinkedInBot
3. Worker reads public/content/blog/index.json (or KV store) to find the post metadata
4. Worker returns a minimal HTML response with per-post og:title, og:description, og:image
5. og:image points to a Cloudflare Images worker URL that renders the branded card (title + category + author) as a PNG
6. LinkedIn displays the branded card — not the generic fallback

Real users get the full React SPA as normal. Only bots get the OG shell.
// workers/og-handler.ts (simplified)
const BOT_AGENTS = /LinkedInBot|Twitterbot|Slackbot|facebookexternalhit/i

export default {
  async fetch(req) {
    const ua = req.headers.get('user-agent') ?? ''
    if (!BOT_AGENTS.test(ua)) return fetch(req) // real user → SPA

    const slug = new URL(req.url).pathname.split('/').at(-1)
    const post = await getPostMeta(slug) // KV lookup
    const ogImage = `https://og.aaryaai.dev/${slug}.png`

    return new Response(`<html><head>
      <meta property="og:title" content="${post.title}">
      <meta property="og:description" content="${post.excerpt}">
      <meta property="og:image" content="${ogImage}">
    </head></html>`, {
      headers: { 'content-type': 'text/html' }
    })
  }
}
