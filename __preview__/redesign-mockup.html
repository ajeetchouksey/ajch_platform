Claude Code Basics & Setup

Claude Code is Anthropic's agentic coding tool that runs directly in your terminal. It has full access to your filesystem, can run commands, create and edit files, and use git — within your explicitly granted permissions.

Permission Model

Claude Code uses a CLAUDE.md file at the project root to define context, preferences, and restrictions. It is NOT stored server-side — it is a plain file in your repo.

🚨
Exam Trap: "Claude Code stores project config in Anthropic's cloud." — FALSE. CLAUDE.md lives in your local repo. Anthropic does not store project-level configuration files.

Workflow Automation with Claude Code

Claude Code can run in headless (non-interactive) mode using --print and --headless flags. This enables CI/CD pipelines to invoke Claude for automated code review, test generation, or documentation updates.

💡
Key mental model: Claude Code is not a "chat assistant in a terminal" — it is an autonomous agent with tool access. Treat it like a junior engineer with sandboxed filesystem access.

Checkpoint & State Management

For long-running tasks, Claude Code supports external checkpointing. When processing hundreds of files, you should save state externally — not rely on context window continuity.

🚨
Exam Trap: "Use a long context window for processing hundreds of items." — It won't fit, degrades mid-task, and vanishes on crash. External checkpoint is the correct answer.

Custom Commands & Slash Commands

Custom slash commands are defined as .md files in .claude/commands/. They can include dynamic placeholders like $ARGUMENTS and be scoped to a project or user-level.
