# Content Schema

This page documents the JSON and Markdown schemas used across the platform's `public/content/` directory.

## Blog Manifest — `blog/index.json`

```json
{
  "posts": [
    {
      "slug": "my-article-slug",
      "title": "Article Title",
      "date": "2026-05-29",
      "tags": ["azure", "ai", "devops"],
      "summary": "Short description shown in the blog card.",
      "readingTime": 5,
      "draft": false
    }
  ]
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `slug` | string | ✅ | URL segment, matches filename `posts/{slug}.md` |
| `title` | string | ✅ | Display title |
| `date` | string (ISO 8601) | ✅ | Publication date |
| `tags` | string[] | ✅ | Topic tags for filtering |
| `summary` | string | ✅ | Card description |
| `readingTime` | number | ✅ | Estimated minutes |
| `draft` | boolean | — | If `true`, excluded from stat counts |

Blog posts are Markdown files at `blog/posts/{slug}.md`. GFM and raw HTML are supported.

## Exam Registry — `exams/index.json`

```json
{
  "exams": [
    {
      "id": "ccaf",
      "name": "Claude Certified Architect · Foundations",
      "shortName": "CCA-F",
      "available": true,
      "questions": 68,
      "questionFiles": ["content/questions/domain1.json"],
      "scenarioFiles": ["content/scenarios/customer-support.json"],
      "domains": [
        {
          "id": 1,
          "name": "Agentic Architecture",
          "weight": 25,
          "notesFile": "content/notes/d1-agentic-architecture.md"
        }
      ]
    }
  ]
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | ✅ | URL segment (e.g., `ccaf`, `ab100`) |
| `available` | boolean | ✅ | If `false`, exam is hidden from catalog |
| `questions` | number | ✅ | Total question count (used in stats) |
| `questionFiles` | string[] | ✅ | Paths to question JSON files |
| `scenarioFiles` | string[] | ✅ | Paths to scenario JSON files |
| `domains[].notesFile` | string | ✅ | Path to study notes Markdown file |

## Question Bank — `questions/*.json`

```json
[
  {
    "id": "q001",
    "domain": 1,
    "question": "Question text here.",
    "options": ["Option A", "Option B", "Option C", "Option D"],
    "answer": 0,
    "explanation": "Why option A is correct.",
    "difficulty": "medium",
    "tags": ["agentic", "orchestration"]
  }
]
```

| Field | Type | Values | Description |
|-------|------|--------|-------------|
| `id` | string | — | Unique within the exam |
| `domain` | number | — | Domain number (matches `domains[].id`) |
| `answer` | number | 0–3 | Index into `options` array |
| `difficulty` | string | `easy`, `medium`, `hard` | Question difficulty |

## Platform Stats — `stats.json`

Auto-generated by `scripts/sync-stats.py`. Do not edit manually.

```json
{
  "schema": "1.0",
  "generated": "2026-07-23",
  "note": "Auto-updated by scripts/sync-stats.py.",
  "platform": {
    "blog_posts": 58,
    "questions": 590,
    "exams": 4,
    "notes": 19,
    "scenarios": 13,
    "agents": 24,
    "tools": 0
  }
}
```

The home page reads this file on mount to display live platform counters. Run `python3 scripts/sync-stats.py` to regenerate after any content update.

## Interview Prep — `interviews/`

The Interview Prep feature uses a canonical bank (dedup model) with per-role packs that reference bank questions and add role-specific addenda.

### Role Index — `interviews/index.json`

```json
{
  "roles": [
    {
      "id": "agentic-ai-platform-architect",
      "title": "Agentic AI Platform Architect",
      "seniority": "Principal",
      "yearsRange": "12–16 yrs",
      "questionCount": 8,
      "accentColor": "violet",
      "description": "...",
      "locations": ["Pune", "Bangalore"],
      "industry": {
        "label": "Enterprise Software",
        "domain": "Platform Engineering",
        "summary": "...",
        "focusAreas": ["Multi-agent orchestration", "SDK design"]
      }
    }
  ]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | URL segment; matches `roles/{id}/pack.json` |
| `seniority` | string | `Senior`, `Principal`, `Staff`, etc. |
| `accentColor` | string | Tailwind color name for UI theming |
| `industry` | object | Optional industry context block |

### Bank Questions — `interviews/bank/questions.json`

```json
[
  {
    "id": "q-orch-001",
    "roles": ["agentic-ai-platform-architect"],
    "competency": "orchestration",
    "type": "technical",
    "difficulty": "senior",
    "question": "...",
    "summary": "...",
    "deepDive": "...",
    "realScenario": "First-person spoken answer starting with \"In my current role...\"",
    "workedExample": { "code": "...", "language": "python", "explanation": "..." },
    "useCases": [],
    "tradeoffs": [],
    "antiPatterns": [],
    "followUps": [],
    "redFlags": []
  }
]
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Globally unique across the bank |
| `roles` | string[] | Which role packs reference this question |
| `competency` | string | Matches `competencies.json` id |
| `realScenario` | string | **First-person** spoken answer; must start with "In my current/previous role" |

### Role Pack — `interviews/roles/:id/pack.json`

```json
{
  "roleId": "agentic-ai-platform-architect",
  "industry": { "label": "...", "domain": "...", "summary": "...", "focusAreas": [] },
  "competencyWeights": [
    { "competencyId": "orchestration", "weight": 30 }
  ],
  "items": [
    {
      "ref": "q-orch-001",
      "addendum": {
        "roleContext": "Why this matters for this specific role...",
        "industryAngle": "Industry-specific consideration...",
        "extraFollowUps": []
      }
    }
  ]
}
```

Items are resolved at runtime by `loadResolvedPackItems()` which merges bank fields with the role-specific addendum. Questions in the bank are never duplicated — each `items[]` entry is a reference (`ref`) not a copy.
