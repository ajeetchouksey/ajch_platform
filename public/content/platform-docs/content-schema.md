# Content Schema

This page documents the JSON and Markdown schemas used across the platform's `public/content/` directory.

## Blog Manifest — `blog/index.json`

```json
{
  "posts": [
    {
      "slug": "my-article-slug",
      "title": "Article Title",
      "date": "2026-05-29",
      "tags": ["azure", "ai", "devops"],
      "summary": "Short description shown in the blog card.",
      "readingTime": 5,
      "draft": false
    }
  ]
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `slug` | string | ✅ | URL segment, matches filename `posts/{slug}.md` |
| `title` | string | ✅ | Display title |
| `date` | string (ISO 8601) | ✅ | Publication date |
| `tags` | string[] | ✅ | Topic tags for filtering |
| `summary` | string | ✅ | Card description |
| `readingTime` | number | ✅ | Estimated minutes |
| `draft` | boolean | — | If `true`, excluded from stat counts |

Blog posts are Markdown files at `blog/posts/{slug}.md`. GFM and raw HTML are supported.

## Exam Registry — `exams/index.json`

```json
{
  "exams": [
    {
      "id": "ccaf",
      "name": "Claude Certified Architect · Foundations",
      "shortName": "CCA-F",
      "available": true,
      "questions": 68,
      "questionFiles": ["content/questions/domain1.json"],
      "scenarioFiles": ["content/scenarios/customer-support.json"],
      "domains": [
        {
          "id": 1,
          "name": "Agentic Architecture",
          "weight": 25,
          "notesFile": "content/notes/d1-agentic-architecture.md"
        }
      ]
    }
  ]
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string | ✅ | URL segment (e.g., `ccaf`, `ab100`) |
| `available` | boolean | ✅ | If `false`, exam is hidden from catalog |
| `questions` | number | ✅ | Total question count (used in stats) |
| `questionFiles` | string[] | ✅ | Paths to question JSON files |
| `scenarioFiles` | string[] | ✅ | Paths to scenario JSON files |
| `domains[].notesFile` | string | ✅ | Path to study notes Markdown file |

## Question Bank — `questions/*.json`

```json
[
  {
    "id": "q001",
    "domain": 1,
    "question": "Question text here.",
    "options": ["Option A", "Option B", "Option C", "Option D"],
    "answer": 0,
    "explanation": "Why option A is correct.",
    "difficulty": "medium",
    "tags": ["agentic", "orchestration"]
  }
]
```

| Field | Type | Values | Description |
|-------|------|--------|-------------|
| `id` | string | — | Unique within the exam |
| `domain` | number | — | Domain number (matches `domains[].id`) |
| `answer` | number | 0–3 | Index into `options` array |
| `difficulty` | string | `easy`, `medium`, `hard` | Question difficulty |

## Platform Stats — `stats.json`

Auto-generated by `scripts/sync-stats.py`. Do not edit manually.

```json
{
  "schema": "1.0",
  "generated": "2026-05-29",
  "note": "Auto-updated by scripts/sync-stats.py.",
  "platform": {
    "blog_posts": 55,
    "questions": 158,
    "exams": 3,
    "notes": 12,
    "scenarios": 9,
    "agents": 21,
    "tools": 9
  }
}
```

The home page reads this file on mount to display live platform counters. Run `python3 scripts/sync-stats.py` to regenerate after any content update.
